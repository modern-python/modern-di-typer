# Change: Open the request-scope command container

**Lane:** lightweight — 1 file, no new file, no public-API change, existing
test suite reproduces (10 of 11 tests failed with `ContainerClosedError`).

## Goal

`_build_command_container` built the per-command `Scope.REQUEST` child
container and closed it on exit, but never opened it. Under modern-di 2.x this
was harmless (resolution worked on an unopened container); under modern-di
3.0's mandatory-open lifecycle, resolving from — or building a further child
of — an unopened container raises `ContainerClosedError`. Every injected
command broke.

## Approach

Mirror `action_scope` (same file), which already opens its child container
with `with ... as container:`. Replace the manual
`build_child_container()` / `try` / `finally: close_sync()` with a `with`
block that opens, yields, and closes the container in one statement — no
behavior change beyond fixing the missing open.

## Files

- `modern_di_typer/main.py` — `_build_command_container` now does
  `with fetch_di_container(ctx).build_child_container(scope=Scope.REQUEST) as container: yield container`.
