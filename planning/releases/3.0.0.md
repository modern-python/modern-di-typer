# modern-di-typer 3.0.0 тАФ modern-di 3.x

Requires **modern-di >= 3, < 4**. **No public API change** тАФ `FromDI`,
`action_scope`, `fetch_di_container`, `inject`, and `setup_di` keep their
signatures and behavior.

## Fix

- **The per-command `Scope.REQUEST` container is now opened before use.**
  `_build_command_container` built the command's child container and closed it
  on exit, but never opened it тАФ under modern-di 2.x this was harmless, but
  modern-di 3.0's mandatory-open lifecycle raises `ContainerClosedError` when
  resolving from (or building a child of) an unopened container, breaking
  every `@inject`-decorated command. Fixed by opening the child with `with`,
  mirroring the pattern `action_scope` already used. See
  [modern-di 3.0's mandatory-open lifecycle](https://modern-di.modern-python.org/migration/to-3.x/).

## Internal refactors

- **Adapted the test suite to a second 3.0 break (test-only, no behavior
  change):**
  - Replaced the removed `Factory(cache_settings=...)` argument with `cache=`
    in `tests/dependencies.py`.
  - Opened the root container with `with modern_di.Container(...) as container:`
    in the `app` fixture (`tests/conftest.py`), per the mandatory-open
    lifecycle.

## Packaging

- **Bumped the `modern-di` dependency to `>=3,<4`.** Requires modern-di 3.x;
  drops support for modern-di 2.x.

## Downstream

Upgrade to pick up the fix тАФ any command using `@inject` was broken under
modern-di 3.0 until this release. No other action needed beyond raising your
own `modern-di` floor to `>=3`. If your own code builds child containers,
open them with `with` / `async with` (or `open()`) before resolving тАФ see the
[modern-di 3.x migration guide](https://modern-di.modern-python.org/migration/to-3.x/).

## Internals

- 100% line coverage; `ruff`, `ty` clean.
