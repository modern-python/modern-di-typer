# modern-di-typer 2.1.1 — release pipeline on PyPI Trusted Publishing

No library changes. The package is identical to 2.1.0; this release exercises the new publish path end-to-end.

## CI

- Releases now authenticate to PyPI via **Trusted Publishing (OIDC)** instead of a long-lived `PYPI_TOKEN` secret. `uv publish` auto-detects the GitHub Actions id-token; the release job runs under a `pypi` environment that scopes the trusted publisher (#21).

## Downstream

No action required. Nothing about the installed package changes.
