# modern-di-typer 2.1.0 — `action_scope` for ACTION-scoped dependencies

Adds a public `action_scope(ctx)` context manager so commands can resolve
`Scope.ACTION` dependencies without injecting `modern_di.Container` and building
a child container by hand. Backward compatible — no action required to upgrade.

## Feature

- **`action_scope(ctx)`.** A context manager that yields a fresh `Scope.ACTION`
  container (a child of the per-command `Scope.REQUEST` container) and closes it
  on exit. Each `with` block is its own action scope, so a single command can
  open as many as it needs — one per unit of work:

  ```python
  @app.command()
  @inject
  def my_command(ctx: typer.Context) -> None:
      for job_name in job_names:
          with action_scope(ctx) as action:
              action.resolve_provider(Dependencies.job).run()
  ```

  A `Scope.REQUEST` singleton resolved inside an action scope is the same
  instance injected into the command — both share the per-command container.

## Downstream

No action needed. The previous pattern — injecting `modern_di.Container` via
`FromDI` and calling `build_child_container()` manually — still works; it is now
the explicit fallback and `action_scope` is the preferred interface.

## Internals

- `@inject` now always builds the per-command `Scope.REQUEST` container and
  stashes it on `ctx.meta` (per-invocation, isolated), so `action_scope` can
  parent the action container onto it.
- Adopted the portable
  [`lesnik512/planning-convention`](https://github.com/lesnik512/planning-convention)
  (`architecture/` + `planning/`), replacing `CONTEXT.md`.
- 100% line coverage across Python 3.10–3.14.
